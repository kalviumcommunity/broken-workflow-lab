# Instructor Notes

## Intended learning outcomes

- Identify weak or missing Git workflow governance in a real repository.
- Design a branching strategy that supports feature work, releases, and hotfixes.
- Define PR requirements, review ownership, and approval rules.
- Specify validation gates and CI expectations before merging changes.
- Describe release strategy, semantic versioning, and rollback readiness.

## Evaluation focus

Evaluate student submissions for:

- completeness of the workflow charter
- clarity of branch naming and purpose
- PR review and approval rules
- validation gate definitions
- release and rollback planning
- alignment with the current repo state and missing governance

## Expected student improvements

Students should create artifacts that:

- document a branch strategy and protect `main`
- describe required PR checks and review roles
- establish how releases are tracked and versioned
- propose rollback readiness and incident-aware updates
- fill gaps in governance without changing app behavior
